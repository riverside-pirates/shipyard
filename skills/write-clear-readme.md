# Write a Clear README

A README is the front door to your project. It's the first thing someone sees
when they visit your repository, and it tells them what your project does, why
it exists, and how to use it. A good README saves people time and shows that
you care about your work.

Writing a clear README is a skill you'll use on every project you build — in
this class and beyond.

## When to Use This Skill

- You're starting a new project and want to set it up right
- Someone asks "what does your project do?" and you want to point them somewhere
- You're improving an existing project that has no README (or a weak one)
- You're preparing to share your work with others

## Step-by-Step Process

### 1. Know Your Audience

Who will read this? A classmate? A teacher? A future employer? Write for them.
If your audience is technical, you can use more jargon. If not, keep it simple.
For class projects, assume your reader is another student in the course.

### 2. Start with What It Is and Why It Exists

Open with one or two sentences explaining what your project does and why you
built it. Don't bury the purpose — lead with it.

### 3. Show How to Get Started

Tell the reader exactly how to install and run your project. Include the actual
commands they need to type. Don't assume they know how — spell it out.

### 4. Explain the Structure

If your project has multiple files or folders, briefly explain what lives where.
This helps people navigate your code without guessing.

### 5. Add Examples

Show what your project looks like in action. Include sample input, sample output,
or a screenshot. Examples make everything click faster than paragraphs of
explanation.

### 6. Keep It Updated

A README that describes a different version of your project is worse than no
README at all. When you change your project, update the README to match.

## Example Scenario

You built a simple Python script called `temp_convert.py` that converts
temperatures between Fahrenheit and Celsius. Here's what a solid README might
look like:

```markdown
# Temperature Converter

A command-line tool that converts temperatures between Fahrenheit and Celsius.
Built for the CS Essentials unit on Python basics.

## How to Run

Make sure you have Python 3 installed, then run:

python temp_convert.py

## Usage

The program will ask you to enter a temperature and choose a conversion
direction. Example:

Enter temperature: 72
Convert to (C)elsius or (F)ahrenheit? C
Result: 22.2°C

## Files

- `temp_convert.py` — the main script
- `README.md` — this file
```

Notice how short and direct it is. That's the goal — enough information to be
useful, nothing extra.

## Tips for a Better README

- **Use headers** (`#`, `##`, `###`) to break up sections. A wall of text is
  hard to scan.
- **Keep paragraphs short.** Two to three sentences is plenty per paragraph.
- **Use code blocks** (triple backticks) for any commands, file names, or code
  snippets. It makes them stand out and easy to copy.
- **Write it early.** Don't wait until the project is "done." Start your README
  when you start your project, and update it as you go.
- **Read it as a stranger.** Pretend you've never seen the project before. Does
  the README make sense? If not, fill in the gaps.

A good README is a sign of a thoughtful developer. Chart your course clearly,
and others will follow.
