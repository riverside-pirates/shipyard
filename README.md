# Shipyard

**The coordination hub for the [riverside-pirates](https://github.com/riverside-pirates) GitHub org.**

Shipyard is where Riverside High School CS students find work to do, track progress, and collaborate across all of our projects. Think of it as the dock where every ship gets built, inspected, and launched.

## How It Works

All of our code lives in separate repos across the `riverside-pirates` GitHub org. Shipyard ties them together. You come here to find **bounties** (tasks to work on), then head to the right repo to build your contribution.

```
 +-----------------+     +-------+     +----------+     +-----------+     +---------+
 | Pick a Bounty   |---->| Build |---->| Dry Dock |---->| Sea-      |---->| Ship It |
 | (find an issue) |     |       |     | (PR      |     | Worthy!   |     |         |
 |                 |     |       |     |  review)  |     | (approved)|     |         |
 +-----------------+     +-------+     +----------+     +-----------+     +---------+
```

1. **Pick a Bounty** -- Browse open issues labeled `bounty` to find something that interests you.
2. **Build** -- Fork or branch, write your code, test it, and commit.
3. **Dry Dock** -- Open a Pull Request. Reviewers (classmates and Mr. B) check your work.
4. **Sea-Worthy** -- Your PR passes review and meets the checklist below.
5. **Ship It** -- Your code gets merged. Nice work, pirate.

## What Are Bounties?

Bounties are GitHub Issues that describe a piece of work that needs to be done. Each bounty includes:

- **A clear description** of what to build or fix
- **Labels** that tell you the difficulty, the repo it belongs to, and the type of work (bug fix, new feature, documentation, etc.)
- **Acceptance criteria** so you know exactly when you're done

To claim a bounty, comment on the issue saying you'd like to work on it. Mr. B or a reviewer will assign it to you. One bounty per person at a time -- finish what you start before grabbing the next one.

## Sea-Worthiness Checklist (Quick Reference)

Before your PR can be merged, make sure you can check all of these off:

- [ ] Code runs without errors
- [ ] You tested it yourself and it does what the bounty asked for
- [ ] No leftover debug code (`console.log`, `print("test")`, commented-out junk)
- [ ] Your commit messages are clear and describe what you changed
- [ ] You followed the style and conventions of the repo you're contributing to
- [ ] PR description explains what you did and links to the bounty (issue)

See [CONTRIBUTING.md](CONTRIBUTING.md) for the full checklist and detailed steps.

## Getting Started

1. **Join the org.** Mr. B will add you to the `riverside-pirates` GitHub org.
2. **Read [CONTRIBUTING.md](CONTRIBUTING.md).** It walks you through everything step by step.
3. **Find a bounty.** Look for issues labeled `bounty` in this repo or any of the project repos.
4. **Ask questions.** If you're stuck, comment on the issue or ask in class. Nobody expects you to know everything already.
